创建我的blog——Hexo+github

发表于 2020-05-23 更新于 2020-05-24

心血来潮想要像其他专业技术人员一样有个自己的博客，记录和分享自己的各种采坑过程，那么就从创建博客开始吧~
基于Mac OS

安装基础环境

官网党可以参考官网进行安装, 上面有详细的步骤,一手资料最靠谱。对于想偷懒的小伙伴也可以直接看参考我的安装步骤

  1. 安装Node.js
    用来生成静态页面,具体安装方式参考Node.js官网,下载mac pkg安装包一路安装即可。
    执行node -v 检查是否安装成功
  2. 安装Git
    用来将本地Hexo内容提交到Github上。这个可以跳过，基本大家都有git了。没有的话可以通过homebrew安装或者装个Xcode

安装Hexo

  1. 安装Hexo
    当Node.js和Git都安装好后就可以正式安装Hexo了，终端执行如下命令：
    $ npm install -g hexo

    注意：
    1、不同版本的mac系统可能会有不同的情况，如果遇到权限不足的问题可以在前面加上sudo
    2、官网上的命令是$ npm install -g hexo-cli,猜测是windows版本的安装命令

  2. 初始化blog
    安装好Hexo之后，就可以初始化blog项目了

    • 在你的本地电脑上创建一个目录，用来存放blog的文章，我的目录名叫Myblog $ mkdir Myblog
    • 进入到Myblog目录中，执行命令: $ hexo init
    • 在Myblog目录下执行命令 $ npm install 安装npm
    • 安装成功后可以看到Myblog多了很多文件和目录
    • 安装完成后，本地blog就算设置成功了，可以通过$ hexo s命令在本地启动服务，预览博客
    • 此时，浏览器中打开网址[http://localhost:4000]，能看到如下页面![ ](/My-New-Post/Hexo4000.png) 此时本地的blog就已经设置好了

关联github仓库地址

本地创建好blog之后，需要与github的blog主页进行关联

  1. 创建仓库

    • 在github官网登录你的github账号，新建仓库，仓库名为github用户名.github.io,此处一定要和你的github的用户名保持一致，否则blog会404
    • 创建好的仓库点击setting进入仓库的设置页面,将分支设置为master，并且找到blog地址，用于之后配置到_config.yml文件中

  2. 在blog项目中配置github地址

    • cd进入Myblog目录中，编辑_config.yml文件

    • 找到文件最后，修改如下部分

    • 你需要将xyyanzi换成你自己的gihub的用户名

  3. 部署到远程github

    • 执行hexo generate或者直接hexo g,用来生成静态页面
    • 部署到远程执行命令hexo deploy或者直接hexo d，如果报错ERROR Deployer not found: git，则需要先安装依赖
    • 在Myblog目录下执行
      $ npm install hexo-deployer-git --save
    • 安装完依赖之后再次执行hexo d，即可开始上传至hithub，上传过程中可能会需要输入github的邮箱和密码
    • hexo d执行成功之后，就可以在浏览器中打开[https://xyyanzi.github.io/] (将xyyanzi换成你自己的)，此时就能看到和[http://localhost:4000]一样的页面了

添加ssh key到Github避免每次都需要输入用户名和密码

创建第一篇文章

blog搭建好了，之后就可以愉快的编写文章了

  1. 创建新文章
    • 进入blog项目目录下，执行hexo new My New Post,其中My New Post就是你的文章名称，执行之后，会source/_post目录下生成一个My-New-Post.md的文件，这时就可以打开这个文件愉快的写你的第一个博客了。
  2. 编辑后发布
    • 编辑完文章之后，保存，然后回到blog目录下，执行命令
      hexo g 生成静态页面
      hexo s 进行本地预览
      hexo d 部署发布到远程
