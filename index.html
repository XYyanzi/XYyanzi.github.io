StringBuilder类: 字符串缓冲区，可以提高字符串的操作效率。(可以看成一个长度可以变化的字符串)，底层也是一个数组，但是没有被final修饰，可以改变长度。

Collection集合

集合就是一种容器，可以用来存储多个数据。
集合和数组的区别:

  • 数组的长度的固定的，集合的长度是可变的。
  • 数组可以存储基本数据类型也可以存储对象，但必须是同一类型的元素。集合只能存储对象，而且对象的类型可以不一致。

List接口的特点:

  • 有序的集合(存储和取出顺序相同)
  • 允许有重复的元素
  • 有索引，可以使用普通的for循环遍历
  • Vector集合、ArrayList集合、LinkedList集合

Set接口的特点:

  • 不允许有重复的元素
  • 没有索引，不能使用普通的for循环遍历
  • TreeSet集合、HashSet集合(无序)<–LinkedHashSet集合(有序)

Collection接口的共用方法

public boolean add(E e) 向集合中添加元素
public void clear() 清空集合中的所有元素
public boolean remove(E e) 把给定的对象在当前集合中删除
public boolean contains(E e) 判断当前集合中是否包含给定元素
public boolean isEmpty() 判断当前集合是否为空集合
public int size() 返回集合中元素的个数
public Oject[] toArray() 把集合中的元素存储到数组中

Iterator迭代器

迭代: 是Collection集合元素的通用获取方式。在取出元素之前先判断集合中是否有元素，如果有就把这个元素取出来，然后继续在判断，直到把集合中元素全部取出。这种取出方式的专业术语就叫迭代。

Iterator接口的两个常用方法
boolean hasNext(E e) 判断集合中是否存在元素
E next() 取出迭代的下个元素

Iterator是一个接口，不能直接创建对象，需要通过实现类来创建对象，它获取实现类对象的方式比较特殊。Collection接口中有个iterator()的方法，这个方法返回的就是Iterator的实现类对象。

迭代器的使用步骤:

  • 使用集合中的iterator()方法获取迭代器的实现类对象，并使用Iterator接口来接收(多态)
  • 使用Iterator接口中的方法HasNext()判断集合中是否存在元素
  • 使用Iterator接口中的方法Next()取出集合中的下一个元素

使用iterator()方法创建实现类对象时，会创建一个指向-1索引的头指针
hasNext()方法用来判断当前指针后面是否还有元素存在
next()方法做了两件事，一是取出下一个元素的值，二是将指针向后移动一位

增强for循环

JDK1.5之后出现的新特性，简化迭代器的for循环书写

final关键字

final关键字代表最终的、不可改变的。

可以修饰一个类: 被修饰的类不能有任何子类

继承

继承主要解决的问题是: 共性抽取。

继承关系的特点:
1、子类可以拥有父类的可继承的”内容”(成员变量和成员方法)
2、子类还可以拥有自己专有的内容

就近原则
在父子类的继承关系中，如果成员变量重名，则创建子类对象时，访问方式有两种:
1、直接通过子类对象访问成员变量:

等号左边是谁，就优先用谁， 没有则向上找

2、间接通过成员方法访问成员变量:

方法属于谁就优先用谁，没有则向上找

子类中的三种变量访问方式:
局部变量 直接写变量名
本类的成员变量 this.变量名
父类的成员变量 super.变量名

方法重写

重写的概念: 在继承关系中，方法名称和参数列表都一样，叫做重写(OverWrite)

重写(OverWrite): 方法名称相同，参数列表也相同
重载(OverLoad): 方法名称相同，参数列表不同

注意事项:

1、必须保证父子类之间方法的名称、参数列表都相同
2、子类方法的返回值范围必须小于等于父类方法的返回值范围
3、子类方法的权限必须大于等于父类方法的权限修饰符

public > protected > (default) > private

继承关系中，父子类构造方法的访问特点:
1、子类构造方法中有一个默认隐含的”super()”调用，因此创建子类对象时，一定是先执行父类的构造方法，再执行子类的构造方法
2、子类构造可以通过super关键字来调用父类重载构造方法
3、super()的父类构造调用，必须是子类构造方法的第一个语句。一个子类只能调用一次super()方法

总结:子类必须调用父类的构造方法，不写则赠送super()，写了就用指定的super调用。super()只能有一个且只能在子类构造方法的第一行

super关键字的三种用法:
1、子类成员方法访问父类的成员变量
2、子类的成员方法访问父类的成员方法
3、子类的构造方法访问父类的构造方法

this关键字的三种用法:
1、本类的成员方法中访问本类的成员变量
2、本类的成员方法中访问本类的另一个成员方法
3、本类的构造方法中访问本类的另一个构造方法

super和this两种构造调用，不能同时使用

java语言是单继承的
java语言可以多级继承
一个子类的直接父类是唯一的，但是一个父类可以拥有多个子类

抽象

抽象的概念:
如果父类当中的方法不确定方法体的具体实现，那么这就应该是一个抽象方法。

抽象类和抽象方法的使用方法:
1、不能直接创建抽象类对象，必须使用一个子类来继承抽象类
2、子类必须覆盖重写父类中所有的抽象方法。（也就是实现）

接口

生活中: 接口就是一种公共的规范标准。
程序中: 接口就是多个类的公共规范。
接口是一种引用类型，最重要的内容就是其中的抽象类方法。

java7的接口内容包括: 常量、抽象方法
java8的接口内容包括: 常量、抽象方法、默认方法、静态方法
java9的接口内容包括: 常量、抽象方法、默认方法、静态方法、私有方法

接口的使用:
1、接口不能直接使用，必须有一个”实现类”来实现该接口
public class 实现类名称 implements 接口名称{}
2、接口的实现类必须覆盖重写接口的所有抽象类
3、创建实现类的对象进行使用

接口的默认方法:
默认方法有方法体，主要为了解决接口升级的问题.
接口的默认方法可以通过接口实现类对象直接调用
接口的默认方法也可以被接口实现类覆盖重写

接口的静态方法:
不能通过实现类对象来调用，只能通过接口名称直接调用 接口名.静态方法()

接口的私有方法:
接口的私有方法只有接口内部可以调用，实现类不可以调用。(主要用于接口内部的静态方法和默认方法调用)
普通私有方法: 提供给接口默认方法调用
静态私有方法: 提供给接口静态方法调用

接口常量的定义和使用:
接口当中的常量: public static final 数据类型 常量名称 = 数据值; 其中（public static final）可以不写, 即使不写也是常量，不可以修改。

类与类之间是单继承的。直接父类只有一个。
类与接口是多实现的。一个类可以实现多个接口。
接口与接口之间是多继承的。

多态

【对象的多态性】:一个对象拥有多个形态，就是多态。

小明是一个学生，但同时也是一个人。
小明是一个对象，这个对象既有学生形态，也有人类形态。

使用格式

【父类引用指向子类对象】
父类名称 对象名 = new 子类名称()
接口名称 对象名 = new 实现类名称()

访问成员变量的两种方式:
1、直接通过对象名称访问: 看创建对象时=左边是谁，优先用谁，没有则向上找
2、间接通过成员方法访问: 看new出来的是谁，就运行谁，没有则向上找

成员变量: 编译看左边，运行也看左边
成员方法: 编译看左边，运行看右边

向上转型

对象的向上转型其实就是多态的写法: 父类名称 对象名 = new 子类名称();
含义： 右侧创建一个子类翠香，把他当做父类来看待
向上转型一定是安全的，因为是从小范围转向了大范围

向下转型

对象的向下转型，其实是一个【还原】动作: 子类名称 对象名 = （子类名称）父类对象;
含义: 将父类对象还原为本来的子类对象

字符串

字符串的特点:
1、字符串的内容永不可变,是一个常量
2、正是因为字符串不可改变，所以字符串是可以共享使用的。
3、字符串效果上相当远char[]字符数组，但是底层原理是byte[]字节数组

创建字符串的方式

三种构造方法，一种直接创建

  • public String(): 创建一个空白字符，不含任何内容
  • public String(char[] array): 根据字符数组的内容来创建字符串
  • public String(byte[] array): 根据字节数组的内容来创建字符串
  • String str = “hello”; 右边直接用双引号

注意:直接写上双引号，就是字符串对象

字符串常量池: 程序中直接写双引号创建的字符串，就在字符串常量池中。 【堆内存中保存, 实际保存的是字节数组的地址, new 创建的字符串不再常量池中。
==判断: 对于基本数据类型，比较的是值，对于引用类型，比较的是地址值

equals()方法

如果要判断两个引用对象的值是否相等，可以使用equals方法:
public boolean equals(Object obj)
注意事项:
1、equals方法具有对称性，a.equals(b) 和b.equals(a)是一样的
2、如果比较的是一个常量和一个变量，推荐将常量写在前面 “q”.equals(b),防止变量的值是null的情况下报空指针错误.
3、equalsIgnoreCase() 方法可以忽略英文大小写

static关键字

被static关键字修饰的成员变量或成员方法，不属于对象自己，而是属于类，本类对象共享。

注意事项:
1、静态不能直接访问非静态
原因：在内存当中【先】有静态内容，【后】有的非静态
2、静态方法中不能使用this
原因：this是只当前对象，但静态方法不需要对象
3、静态优先于非静态，因此静态代码块比构造函数先执行

静态代码块的典型用途: 用来一次性的对静态变量进行赋值

静态变量和静态方法在内存中是在方法区的静态区中。

类和对象

面向对象三大特征:

  • 封装
  • 继承
  • 多态

类: 是一组相关属性和行为的集合。可以看成是一类事物的模板，使用事物的属性特征和行为来描述该类事物
对象：是一类事物的具体实现。对象是类的一个实例，必然具备该类事物的属性和行为。

类是对一类事物的描述，是抽象的。
对象是一类事物的实例，是具体的。
类是对象的模板，对象是类的实体。

类的定义

类的使用

通常情况下，一个类并不能直接使用，需要根据类创建一个对象才能使用。

  • 导包
    • 指出需要使用的类的所在位置 import 包名称.类名称
    • 如果和要使用的类在相同的包下，可以胜率导包语句
  • 创建
    • 类名称 对象名 = new 类名称()
    • Student stu = new Student()
  • 使用
    • 使用成员变量 对象名.成员变量
    • 使用成员方法 对象名.成员方法(参数)

如果成员变量没有进行赋值，name将会有一个默认值，规则和数组一样。

new出来的对象包括成员变量和成员方法都是存储在堆内存中
堆内存中存储的是成员方法在方法区的地址值
执行成员方法时，也会进入到栈内存中，成员方法执行结束后会立即出栈


成员变量和局部变量的区别

  • 定义的位置不一样
    • 局部变量: 方法内部
    • 成员变量: 方法外部，直接写在类里
  • 作用范围不一样
    • 局部变量: 只有方法中可以使用，出了方法就不能再使用
    • 成员变量: 整个类都可以使用
  • 默认值不一样
    • 局部变量: 没有默认值，需要手动进行赋值
    • 成员变量: 有默认值，规则和数据一样
  • 内存位置不一样
    • 局部变量: 栈内存
    • 成员变量: 堆内存
  • 生命周期不一样
    • 局部变量: 随着方法进栈而诞生，随着方法出栈而消失
    • 成员变量: 随着对象创建而诞生，随着对象被垃圾回收而消失

封装性

封装性在java中的体现
1、方法就是一种封装
2、关键字private也是一种封装

private关键字修饰需要保护的成员变量。被private修饰的成员变量，只能在本类中进行访问，超出了本类范围之外就不能再直接访问(类名.成员变量的方式)了。

本类范围外想访问被private修饰的成员变量: 通过getter、setter方法
对于基本类型boolean类型的成员变量，在getter时，方法名应该是isXx()

this关键字: 当方法的局部变量和类的成员变量重名的时候，根据”就近原则”，优先使用局部变量，如果需要访问本类当中的成员变量，需要使用this关键字
通过谁调用的方法，谁就是this，this就代表类本身

构造方法

构造方法是专门用来创建对象的方法,当我们通过关键字new来创建对象时，其实就是在调用构造方法。 （new对象的时候实际就是在调用构造方法）

  • 构造方法的名称和类名称必须一样，大小写也要一样
  • 构造方法不能写返回值类型
  • 构造方法不能return一个具体的返回值
  • 如果没有编写任何构造方法，那么编译器会默认创建一个空的构造方法
  • 一旦自己编写了至少一个构造方法，编译器就不会默认创建构造方法了

一个标准的类的组成:

  • 所有的成员变量都要使用private关键字进行修饰
  • 为每一个成员变量编写一对Getter/Setter方法
  • 编写一个无参数的构造方法
  • 编写一个全参数的构造方法

这样标准的类也叫做Java Bean

数组的定义

数组: 是一种容器，可以同时存放多个数据值，存放的数据类型必须统一
数组的特点:
  1. 数组是一种引用数据类型
  2. 数组当中的数据，类型必须统一
  3. 数组的长度在程序运行期间不可以改变

数组的初始化: 在内存中创建一个数组，并且向其中赋予一些默认值
两种初始化方式:
  1.动态初始化(指定长度)
  2.静态初始化(指定内容)

动态初始化格式:
数据类型[] 数组变量名称 = new 数据类型[数组长度]

静态初始化格式:
数据类型[] 数组变量名 = new 数据类型[]{元素1，元素2，…};
数据类型[] 数据变量名 = {元素1，元素2，…};

数组的使用

获取数组中的值: 数组名[索引值]
索引值从0开始，到“数组长度-1”为止

注意事项：

使用动态初始化数组时，其中的元素会自动拥有一个默认值

  • 如果是整数类型，默认0
  • 如果是浮点类型，默认0.0
  • 如果是字符类型，默认‘\u0000’
  • 如果是布尔类型，默认是false
  • 如果是引用类型，默认是null

java内存

Java的内存需要划分成为5个部分:

  • 栈内存(Stack): 存放的都是方法中的局部变量。方法的运行一定要在栈当中。

    局部变量: 方法的参数，或者是方法()内部的变量，一旦超出作用域，立即从栈内存中消失

  • 堆内存(Heap): 凡是New出来的东西，都在堆内存中。

    堆内存里面的东西都有一个地址值: 16进制
    堆内存里面的数据都有默认值

  • 方法区(Method Area): 存储.class相关信息，包含方法的信息。

  • 本地方法栈(Native Method Stack): 与操作系统相关。

  • 寄存器(pc Register): 与CPU相关。

数组的内存图

1、运行.class文件时，先从方法区中获取class信息
2、读取到main方法后，会在栈中开辟一个空间，供main方法运行，这个过程叫做进栈
3、定义一个变量array，就是在栈中创建一个局部变量
4、new数组的时候，就是在堆内存中开辟1块空间，该空间有对应的地址值，该地址值会赋值给变量。同时每个数组索引对应的空间中存放默认值
5、输出变量array时，实际输出的是数组的地址值
6、给array[1]赋值时，实际是将原数组索引1的位置的值由默认值替换成实际值

4、 变量的使用不能超过作用域的范围

数据类型转换:
自动类型转换（隐式）：数据范围从小到大 int–>long byte–>int float–>double long–>float
强势类型转换（显式）：数据范围从大到小，精度会发生损失 int num = (int)100L;
注意事项：

2、 byte/short/char可以发生数学运算，例如加法“+”，这三种类型的数据在运算时会首先被转换成整型，然后再进行计算

三种等待方式

    强制等待，线程休眠一段时间 Thread.sleep(5)

  • 隐式等待

    设置一个等待时间，轮询查找(默认时间0.5s)元素是否出现，如果不出现抛出异常 dirver.manage().timeout().implicitlyWait(5, TimeUnit.SECONDS)

  • 显示等待

多窗口切换

多窗口切换通过切换句柄实现
1、先获取到当前的窗口句柄 driver.getWindowHandle()
2、再获取到所有的窗口句柄 driver.getWindowHandles()
3、判断是否是想要操作的窗口，如果是，继续操作，如果不是，进行切换之后再操作 driver.switchTo().window(name)

frame切换

web自动化中如果一个元素定位不到，name很大可能是在iframe中。
frame是html中的框架，在html中所谓的框架就是可以再同一个浏览器中显示不止一个页面。
frame分为 frameset、frame、iframe三种
frame和iframe对selenium定位而言是不样的，selenium有一组专门的api对frame进行操作。
frame存在两种，一种是嵌套的，一种是未嵌套的。

切换frame
driver.switchTo().frame() 根据元素id或者index切换frame
driver.switchTo().defaultContent() 切换到默认frame
driver.switchTo().parentFrame() 切换到父级frame

对于未嵌套的iframe可以直接使用id进行定位，也可以使用index进行定位
对于嵌套的iframe先进入到iframe的父节点，再进到子节点，然后可以对子节点里面的对象进行处理和操作
